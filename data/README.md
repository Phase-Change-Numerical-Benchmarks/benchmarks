# Reference Data

Each benchmark stores its reference data under `data/<ID>/reference.csv`.

Conventions:

- plain CSV, comma-separated, LF line endings, one header row,
- column names are lowercase snake_case; units follow the case file,
- analytical and semi-analytical references are generated by
  `scripts/plot_reference_figures.py` and must not be edited by hand
  (CI regenerates them and fails on any mismatch),
- experimental or externally digitized data include their provenance in
  the corresponding case file under "Reference Data",
- long-format records: some cases use a leading `record` column to mix
  scalar constants (e.g. `lambda`, `B_M`) and tabulated profiles in a
  single file; consumers should filter on `record`.
